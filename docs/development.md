# Development

## Repository Goals

- maintain clean structure
- avoid monolithic framework architecture
- prefer standalone modules
- keep infrastructure reproducible

---

## Recommended Workflow

1. Create feature branch
2. Implement changes
3. Test locally
4. Open pull request

---

## Naming Conventions

Repositories:

```txt
redm-*
```

Resources:

```txt
resource-name
```

---

## Philosophy

Infrastructure and gameplay systems should remain separated whenever possible.
