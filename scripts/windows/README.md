# Windows Server Scripts

Windows PowerShell scripts are the current working script-based runtime flow.

Run all commands from the repository root.

---

## Scripts

| Script | Responsibility |
|---|---|
| `scripts/windows/01-server-install.ps1` | Downloads and extracts Windows FXServer artifacts into `server/` |
| `scripts/windows/02-server-setup.ps1` | Creates runtime directories, copies templates/assets, and installs resource layers |
| `scripts/windows/03-server-run.ps1` | Starts FXServer from `txData/<profile>.base/` and writes logs |

---

## Requirements

- PowerShell
- Git
- 7-Zip available as `7z` or `7za`, or installed in a standard 7-Zip path
- A direct Windows FXServer `.7z` artifact URL

---

## Default Profile

If `-Profile` is not provided, scripts use:

```text
dev
```

The default runtime profile directory is:

```text
txData/dev.base/
```

---

## Runtime Structure

Setup creates the stable txAdmin-compatible control directory:

```text
txData/default/
txData/default/data/
txData/default/logs/
txData/default/config.json
```

`txData/default/config.json` is generated by `02-server-setup.ps1` if missing. Current generated content:

```json
{
  "version": 2
}
```

The selected script-managed runtime profile is created under:

```text
txData/<profile>.base/
```

For the default profile:

```text
txData/dev.base/
```

Runtime profile contents:

```text
txData/<profile>.base/cache/
txData/<profile>.base/resources/
txData/<profile>.base/server.cfg
txData/<profile>.base/resources.cfg
txData/<profile>.base/trembita-resources.cfg
txData/<profile>.base/permissions.cfg
txData/<profile>.base/secrets.cfg
txData/<profile>.base/server-icon.png
```

The setup script does not create `txData/admins.json`.

---

## Step 1: Install FXServer Artifacts

```powershell
./scripts/windows/01-server-install.ps1 -ArtifactUrl "PASTE_ARTIFACT_DOWNLOAD_URL_HERE"
```

This recreates:

```text
server/
```

and verifies:

```text
server/FXServer.exe
```

`server/` is shared by all runtime profiles.

---

## Step 2: Generate Runtime and Install Resources

Default `dev` profile:

```powershell
./scripts/windows/02-server-setup.ps1
```

Explicit profile:

```powershell
./scripts/windows/02-server-setup.ps1 -Profile test
```

This always creates or preserves:

```text
txData/default/
```

and creates the selected runtime profile:

```text
txData/test.base/
```

Config templates are copied from:

```text
config/templates/
```

Resource layers are installed into:

```text
txData/<profile>.base/resources/
```

Layer order:

1. `https://github.com/Trembita-Games/redm-server-data-cfx.git`
2. `https://github.com/Trembita-Games/redm-server-data.git`

Existing generated configs are preserved unless `-Overwrite` is used.

---

## Step 3: Configure Secrets

Edit:

```text
txData/<profile>.base/secrets.cfg
```

For the default profile:

```text
txData/dev.base/secrets.cfg
```

Set:

```cfg
sv_licenseKey "your-license-key"
```

---

## Step 4: Start FXServer

Default `dev` profile:

```powershell
./scripts/windows/03-server-run.ps1
```

Explicit profile:

```powershell
./scripts/windows/03-server-run.ps1 -Profile test
```

The run script starts:

```text
server/FXServer.exe
```

from this working directory:

```text
txData/<profile>.base/
```

and executes:

```text
server.cfg
```

This working directory is required so nested `exec` paths resolve correctly:

```cfg
exec resources.cfg
exec trembita-resources.cfg
exec permissions.cfg
exec secrets.cfg
```

---

## Logs

Runtime logs are written to:

```text
txData/default/logs/<profile>/
```

Log file names use:

```text
server-yyyy.MM.dd-HH.mm.ss.log
```

Example:

```text
txData/default/logs/dev/server-2026.05.18-14.30.00.log
```

---

## Cache

FXServer cache is profile-specific:

```text
txData/<profile>.base/cache/
```

---

## Direct FXServer Startup

For the script-based workflow, use `03-server-run.ps1` instead of launching `server/FXServer.exe` from the repository root.

Direct startup from the wrong working directory can break nested `exec` resolution and place runtime files outside the expected profile directory.

Equivalent direct startup for the default profile:

```powershell
Push-Location txData/dev.base
../../server/FXServer.exe +exec server.cfg
Pop-Location
```

## Common Issues

If PowerShell blocks script execution, run with an execution policy override:

```powershell
powershell -ExecutionPolicy Bypass -File ./scripts/windows/01-server-install.ps1 -ArtifactUrl "PASTE_ARTIFACT_DOWNLOAD_URL_HERE"
```

If 7-Zip is missing, install 7-Zip and make sure `7z.exe` is available in `PATH`.

If `server/FXServer.exe` is missing, run step 1.

If `txData/<profile>.base/server.cfg` is missing, run step 2 for that profile.
